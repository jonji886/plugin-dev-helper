# 一个例子

### 1、创建 manifest.json 文件

这是插件的配置文件，指定了插件的名称、版本、用户界面文件和虚拟机代码文件。

```json
{ "name": "我的插件", "version": "1.0.0", "frame": "ui.html", "main": "vm.js" }
```

### 2. 创建用户界面文件 `ui.html`

这是插件的用户界面，运行在 `iframe` 中。用户可以通过这个界面与插件进行交互；文件名不限只要保证是HTML文件。

```html
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>
<body>
<h1>我的插件</h1>
<button id="getDesignJsonBtn">获取方案JSON(点击后等待几秒)</button>
<pre id="jsonContent"></pre>
<script>
// 监听来自虚拟机代码的消息
window.addEventListener("message", (event) => {
  if (event.data.action === "designJsonUrl") {
    // 读取文件内容并展示
    fetch(event.data.value)
      .then(response => response.json())
      .then(json => {
        document.getElementById("jsonContent").innerText = JSON.stringify(json, null, 2);
      })
      .catch(error => {
        document.getElementById("jsonContent").innerText = `Error: ${error.message}`;
      });
  }
});
// 发送消息给虚拟机代码
document.getElementById("getDesignJsonBtn").addEventListener("click", () => {
  window.parent.postMessage({ action: "getDesignJsonUrl" }, "*");
});
</script>
</body>
</html>
```

### 3. 创建虚拟机代码文件 `vm.js`

这是运行在插件虚拟机中的代码，可以调用酷家乐的工具插件API。它负责处理来自用户界面的消息，并将结果发送回用户界面；文件名不限只要保证是JS文件。

```javascript
IDP.Miniapp.view.defaultFrame.mount(IDP.Miniapp.view.mountPoints.main); //默认挂载点,固定值
IDP.Miniapp.view.setContainerOptions(IDP.Miniapp.view.defaultFrame,{position:{x:0,y:0}});//窗口左上角位置
const {windowHeight,windowWidth} = IDP.UI.computeLayout()//当前工具页面布局数据
IDP.Miniapp.view.defaultFrame.resize(windowWidth, windowHeight);//窗口尺寸；跟当前工具窗口一致
IDP.Miniapp.view.setContainerOptions(IDP.Miniapp.view.defaultFrame, { minimizable:true});//窗口支持最小化
// 监听来自用户界面的消息
IDP.Miniapp.view.defaultFrame.onMessageReceive((data) => {
  if (data.action === 'getDesignJsonUrl') {
    // 调用 getDesignFullJsonUrlAsync 获取设计文件的URL
    IDP.Custom.Design.Export.getDesignFullJsonUrlAsync().then((response) => {
      const url = response.data.url;
      // 将设计文件的URL发送回用户界面
      IDP.Miniapp.view.defaultFrame.postMessage({
        action: 'designJsonUrl',
        value: url
      });
    }).catch(error => {
      IDP.Miniapp.view.defaultFrame.postMessage({ 
        action: 'designJsonUrl', 
        value: `Error: ${error.message}` 
      });
    });
  }
});
```

### 说明

- **用户界面 (`ui.html`)**：
  - 一个简单的HTML页面，包含一个按钮。
  - 按钮点击时，通过 `window.parent.postMessage` 向虚拟机代码发送一个消息，请求设计文件的URL。
  - 通过 `window.addEventListener("message", ...)` 监听来自虚拟机代码的消息，读取文件内容并展示。

- **虚拟机代码 (`vm.js`)**：
  - 通过 `IDP.Miniapp.view.defaultFrame.onMessageReceive` 监听来自用户界面的消息。
  - 如果收到的消息类型为 `getDesignFullJsonUrlAsync`，则调用 `IDP.Custom.Design.Export.getDesignFullJsonUrlAsync` 获取设计文件的URL。
  - 将设计文件的URL发送回用户界面。

- **manifest.json 文件**：插件的配置文件，指定了插件的名称、版本、用户界面文件和虚拟机代码文件。

# 工具插件核心组成（2文件1配置）

### 1、用户界面(UI)文件

一个标准的 HTML文件，以一个 `iframe` 显示并挂载到酷家乐云设计工具中；

##### 具备的能力

- 访问标准DOM接口
- 发起网络请求(注意：此时网络请求是跨域的)
- 通过 `window.parent.postMessage` 发送消息至 `VM代码`
- 通过 `window` 的message事件接受来自于 `VM代码` 的消息

##### 注意事项

- 无法直接调用 [工具插件API](https://www.kujiale.com/op/api/doc/redirect?doc_id=1498&language=zh)。
- 可以发起网络请求，但需要被请求的服务支持跨域；参见 [插件如何处理跨域调用](https://www.kujiale.com/op/api/doc/redirect?doc_id=1862&language=zh)
- `iframe` 的src会来自于一个独立的域名，因此，会受到同源策略的限制，无法任意访问父页面的元素和接口

### 2、虚拟机代码(VM)文件

在酷家乐虚拟机(Virtual Machine)中运行的 JavaScript 代码，只有这部分代码可以调用 [工具插件API](https://www.kujiale.com/op/api/doc/redirect?doc_id=1498&language=zh)。

##### 注意事项

###### 对象的赋值

- 酷家乐提供的 **插件接口返回的对象都是只读** 的，并且已经被Object.freeze处理，直接赋值和修改不会生效，在严格模式下会报错。因此在使用酷家乐插件接口返回的对象时，注意不要随意直接赋值修改。如有修改需求，查看接口或文档查看该对象是否提供了一系列修改的方法，如未提供，请联系我们提出诉求。

###### javascript支持情况

- 不支持 `setTimeout` 和 `setInterval` 等延迟执行函数
- 不支持浏览器API和DOM，不支持发起网络请求；
- 因为安全性需要，在生产环境 `console` 相关接口会被替换为空实现；所以不支持在生产环境VM直接console输出日志等信息，建议把需要输出的内容通过 [postMessage](https://manual.kujiale.com/idp-sdk/latest/apis/idp-sdk.unexported.framehost.postmessage) 发送给UI页面去输出；
- 对于Error对象，建议直接log其 `message`、`stack` 属性，直接log整个error可能不能显示所有信息
- 因为插件运行于独立的沙箱环境中，当使用 `Promise` 相关的函数时，会造成运行时上下文发生变化，导致 `Promise` 中产生的错误无法被直接观测到，请在需要的时候捕获异常进行处理;

### 3、manifest.json配置

manifest.json 文件是每个插件必须包含的配置文件，酷家乐平台通过这个配置来获取到插件的VM文件和UI文件。下面是 manifest.json配置的内容示例：

```json
{ "name": "我的插件", "version": "1.0.0", "frame": "page.html", "main": "code.js" }
```

- `name`: 用来指定插件的名称【平台侧不做校验，可自定义填写】
- `version`: 用来指定插件的版本【平台侧不做校验，可自定义填写】
- `frame`：用来指定用户界面要加载的 HTML 文件路径，例如：`frame: "./ui.html"`
- `main`：用来指定VM代码文件。如：`main: "./main.js"`

### 用户界面(UI) 和 虚拟机代码(VM) 的通信示例

##### 用户界面(UI) => 虚拟机代码(VM)

**用户界面(UI)**

```javascript
window.parent.postMessage({ action: "getDesignId" }, "*");// 插件页面向插件代码发送自定义消息
```

**虚拟机代码(VM)**：[onMessageReceive接口文档](https://manual.kujiale.com/idp-sdk/latest/apis/idp-sdk.unexported.framehost.onmessagereceive)

```javascript
IDP.Miniapp.view.defaultFrame.onMessageReceive(data => { //监听message消息，其中IDP.Miniapp.view.defaultFrame指的是默认的插件页面iframe
  if (data.action === 'getDesignId') {// 自定义消息相应类型
    IDP.Miniapp.view.defaultFrame.postMessage({
      action: 'designId',
      value: IDP.Design.getDesignId()
    })
  }
});
```

##### 虚拟机代码(VM) => 用户界面(UI)

**虚拟机代码(VM)**：[postMessage接口文档](https://manual.kujiale.com/idp-sdk/latest/apis/idp-sdk.unexported.framehost.postmessage)

```javascript
IDP.Miniapp.view.defaultFrame.postMessage({ action: 'designId', value: "xxx"})// 向插件页面发送自定义消息，其中IDP.Miniapp.view.defaultFrame指的是默认的插件页面iframe
```

**用户界面(UI)**

```javascript
window.addEventListener("message", event => {//在插件页面中添加事件监听
  if (event.data.action === "designId") {
    // 自定义消息相应类型
    ...
  }
});
```

# 常见问题FAQ

**Q：在用户界面(UI)（html的js 或 vue 或 react等）中调用了 插件API，为什么会报错？**

A：插件API必须写在虚拟机代码(VM)，这部分代码是运行在一个封闭的虚拟机环境，跟浏览器环境不同。
