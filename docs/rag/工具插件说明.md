## 简介

酷家乐历来都坚持开放，我们相信联合全行业力量，是促进整个行业发展的最好的方式。在这个理念的驱动下，酷家乐对开发者生态的战略进行了全面的升级，推出了工具插件（以下简称插件）的整体解决方案。通过插件，酷家乐将联合更多伙伴，将众多现有的、新的产品功能以插件的方式集成到酷家乐这个平台上，共同为行业提供服务。

在用户界面上，插件和工具之间有着紧密的联系，插件自己的 UI 会作为一个窗口运行在工具内。

插件本身大体可分为两部分，页面和 VM。

- VM 指虚拟机。这部分代码可以访问工具提供的 API，试插件以编程的方式使用工具的能力，例如操控场景中的设计对象，改变工具原有的 UI 等。
- 页面部分相当于一个普通的浏览器页面，页面有与酷家乐不同的单独的域名。在页面中运行着插件的交互界面。此外，在页面中可以正常地使用浏览器 API，例如访问文件、发起网络请求（注意请求会是跨域的需要后端处理）等。这让插件能以多种方式与其他系统集成。

> 详情参见：[插件代码构成](https://www.kujiale.com/op/api/doc/redirect?doc_id=1496&language=zh)

## 开放的能力

云设计工具开放多种能力以被插件使用。

**注：工具插件正处于初期阶段，所拥有的能力并不完善。下面提到的能力可能还不健全，可供参考。**

**一、设计对象的操作：创建、读取、编辑、删除。**

这是最核心的能力。开发者可以用编程的方式，操控设计工具里的各类设计对象：墙、柱、门窗、硬装铺贴、成品家具、定制家具等。

这个能力的使用范围相当广泛，例如：

- 各类快速设计的功能，插件可以有自己设计逻辑，利用创建对象的能力，将结果给到云设计工具。
- 不少设计对象背后都有很深的行业/工业领域的属性，例如全屋定制的柜子包含着加工工艺的信息。使用这些信息，可以开发特定行业的个性化功能。

**二、界面自定义。**

界面自定义支持以下几种方式：

- 通过 UI 类的 API，插件可以控制工具原生的界面元素（顶部栏、左侧栏等）。包括简单的整体显示隐藏，以及界面元素本身的可配置化功能（比如自定义菜单）。
- 添加新的界面元素，内容完全由插件决定，数据的来源可以是工具本身或者其他外部系统。

**三、监听工具的事件。**

插件可以监听到工具中的一些事件，响应式地执行相应的动作。例如：

- 场景中某个对象被选中。插件监听到此事件后，可以在自己的 UI 上显示此对象的详细信息。
- 在场景中单击、双击鼠标。这个事件会提供点击的坐标，插件可以再得到坐标后对离这个坐标最近的对象执行一些操作。
- 场景中添加、删除了某个对象。

**四、Serverless**

为了降低开发复杂度，酷家乐提供了一定的 Serverless 能力：

- 对象存储的 BaaS 能力。可提供基本的云端存储能力。
- FaaS。可以访问对应的 BaaS，提供服务器 API。

## 插件的生命周期

插件有着严格的生命周期管理，同一时间只能有一个插件运行。在插件的生命周期从结束后：

- 被插件更改的工具的 UI 状态会恢复到插件启动前的形态。
- 插件的代码会被卸载，做的事件监听也不会在生效。

插件不能自己启动自己，只能被其他角色显式地启动。典型的启动方式是由用户主动触发启动。用户有两个启动插件的入口：顶部栏和对象选择菜单。

## 典型场景

### 自定义素材栏

利用「界面自定义」的能力，开发者可以定制素材栏。一个典型的自定义素材栏通过两步实现：

1. 建立素材栏中要展示的商品和酷家乐模型的对应关系。
2. 利用插件方式开发素材栏的 UI。在点击商品时调用开始摆放模型的 API，API 所需的模型 ID 利用上面的对应关系找到。

### 打通订单

开发者可以做一个插件在云设计工具中直接提交订单，设计师本人的信息可以自动从酷家乐提取。填写完其他信息后，整个表单的数据可以通过网络请求的方式直接发送到外部的订单系统中。

使用这样的订单插件，设计师无需跳出工具即可完成订单的提交，这使得整个业务流程更加高效。

## 与内嵌工具的结合

在原先 Open API 的开发框架下，云设计工具原本就提供了以 `iframe` 的方式内嵌到其他系统的能力。插件可以结合这种方式一起使用，并提供插件与外部系统在前端直接通信的渠道。
